# Contributing to EUSurvey

Thank you for your interest in contributing to EUSurvey.

EUSurvey is an open-source project maintained by the European Commission.
We welcome contributions that improve quality, security, performance,
accessibility, interoperability, and usability.

Please read this document carefully before submitting contributions.


## 1. Code of Conduct

Contributors are expected to:

- Be respectful and constructive.
- Focus on technical merit.
- Avoid personal criticism.
- Follow responsible disclosure for security issues (see SECURITY.md).

By participating in this project, you agree to uphold professional and respectful collaboration standards.


## 2. How to Contribute

You can contribute in several ways:

- Reporting bugs
- Suggesting improvements
- Improving documentation
- Submitting code enhancements
- Proposing new features

Before starting significant work, we recommend opening an issue
to discuss the proposal and confirm alignment with the roadmap.


## 3. Reporting Issues

When reporting a bug, please include:

- Clear description of the issue
- Steps to reproduce
- Expected behaviour
- Actual behaviour
- Screenshots/logs (if relevant)
- Version number (if known)

Security vulnerabilities must NOT be reported via public issues.
Please follow the process described in SECURITY.md.


## 4. Submitting Code Contributions

### Step 1 – Fork the repository

Create a fork of the project under your own namespace.

### Step 2 – Create a feature branch

Use a descriptive branch name, for example:

- `feature/improve-export-format`
- `bugfix/mobile-rendering-issue`
- `docs/update-installation-guide`

### Step 3 – Follow Coding Standards

- Keep changes minimal and focused.
- Ensure your code builds successfully.
- Follow existing project structure and conventions.
- Add comments where appropriate.
- Avoid introducing new dependencies unless justified.

### Step 4 – Tests

If applicable:

- Add or update unit tests.
- Ensure existing tests pass.
- Avoid breaking existing functionality.

### Step 5 – Submit a Merge Request

When ready:

- Provide a clear title and description.
- Reference related issue numbers.
- Explain the rationale for the change.
- Highlight any backward compatibility impact.

All merge requests are subject to review by project maintainers.


## 5. Licensing and Legal Requirements

By submitting a contribution, you agree that:

- Your contribution is your original work, or you have the right to submit it.
- It can be licensed under the European Union Public Licence (EUPL) v1.2.
- You grant the European Commission the right to redistribute your contribution under the project license.

If required, contributors may be asked to confirm this explicitly.


## 6. Acceptance Criteria

A contribution may be accepted if:

- It aligns with the project roadmap.
- It respects architectural principles.
- It does not introduce security risks.
- It maintains backward compatibility (unless justified).
- It passes CI/CD validation.

The maintainers reserve the right to reject contributions
that do not align with the strategic direction of the project.


## 7. Roadmap Alignment

EUSurvey is used by EU Institutions, Bodies, and Agencies.
Feature requests should ideally:

- Improve interoperability
- Enhance accessibility (WCAG compliance)
- Strengthen security and data protection
- Improve scalability and cloud-readiness
- Support EU public sector needs

Commercial or highly specialised custom features
may require separate governance arrangements.


## 8. Development Environment

Basic development guidelines:

- Use the documented build process.
- Do not commit secrets or credentials.
- Use environment variables for configuration.
- Keep dependencies up to date.

Refer to the README.md for setup instructions.


## 9. Documentation Contributions

Documentation improvements are highly encouraged.
Clear documentation increases adoption and sustainability.


## 10. Questions

If you are unsure whether a contribution is suitable,
please open a discussion or issue before investing significant development effort.


---

Thank you for helping improve EUSurvey.
